# Claude Design

**Summary**: Anthropic Labs product for collaborative visual work — designs, prototypes, slides, and more — powered by Claude Opus 4.7 with a design-to-code handoff to Claude Code.

**Sources**: raw/Introducing Claude Design by Anthropic Labs.md

**Last updated**: 2026-04-18

---

Launched April 18, 2026. Available in research preview for Claude Pro, Max, Team, and Enterprise subscribers (included in plan limits, with extra usage available). For Enterprise, off by default — admins enable it in Organization settings. Powered by Claude Opus 4.7, described as Anthropic's most capable vision model.

## What it does

Claude Design follows a natural creative flow: describe what you need → Claude builds a first version → refine through conversation, inline comments, direct edits, or custom sliders (generated by Claude on the fly). The core value proposition is widening exploration — designers can prototype a dozen directions instead of rationing to a few; non-designers (founders, PMs, marketers) can produce visual work without a design background.

## Key capabilities

- **Design system integration**: During onboarding, Claude reads your codebase and design files to build a team design system — colors, typography, components. Applied automatically to every subsequent project. Teams can maintain multiple design systems.
- **Import from anywhere**: Text prompt, uploaded DOCX/PPTX/XLSX, codebase, or web capture (grab elements directly from your live website so prototypes look like the real product).
- **Fine-grained refinement**: Inline comments on specific elements, direct text edits, adjustment knobs for spacing/color/layout — then ask Claude to propagate changes across the full design.
- **Collaboration**: Organization-scoped sharing — private, view-only link, or edit access with group conversation with Claude.
- **Export**: Internal URL, folder, Canva, PDF, PPTX, or standalone HTML.
- **Handoff to Claude Code**: When a design is ready to build, Claude packages it into a handoff bundle passable to Claude Code with a single instruction. Closes the design→implementation loop.

## Use cases

- **Interactive prototypes**: Static mockups → shareable interactive prototypes without code review or PRs
- **PM wireframes**: Sketch feature flows, hand off to Claude Code or designers
- **Design explorations**: Wide range of directions quickly
- **Pitch decks**: Rough outline → complete on-brand deck, export as PPTX or send to Canva
- **Marketing collateral**: Landing pages, social assets, campaign visuals
- **Frontier design**: Code-powered prototypes with voice, video, shaders, 3D, and built-in AI

## Connection to the broader Claude ecosystem

The Claude Code handoff is the most architecturally interesting feature: designs become structured artifacts that an agent can consume directly, rather than requiring manual translation from design to spec to implementation. This fits the pattern in [[agent-harness]] of making artifacts machine-readable at handoff points — here between the design tool and the coding agent.
